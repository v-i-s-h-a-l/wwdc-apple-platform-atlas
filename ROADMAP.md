# Roadmap

This roadmap keeps the repository focused on Apple platform and WWDC research for AI agents.

## Now

- Preserve the WWDC25 and WWDC26 corpus as a public, source-linked routing atlas.
- Keep `llms.txt`, `agent-router.json`, and `agent-corpus/` useful as first-hop agent entry points.
- Keep public validation strict: no full transcripts, no private paths, no credentials, no local helper commands.
- Keep the public reader available through GitHub Pages.

## Next

- Promote more metadata-only sessions into deep-researched records, starting with high-impact developer workflows.
- Improve Apple documentation links for exact API verification.
- Add stronger concept routes for API names, framework names, and workflow questions.
- Add link freshness checks for Apple session and documentation URLs.
- Align the corpus manifest and router output with the generic Agentic Research Router schemas.
- Improve the public reader without making it the primary agent surface.

## Later

- Add release-to-release maps for Apple platform changes.
- Add SDK/API verification notes for beta-to-GM changes.
- Add per-framework "what changed" rollups with source-backed confidence levels.
- Add retrieval-quality checks for common agent questions.
- Add lightweight examples that combine multiple Apple technologies into implementation journeys.

## Non-Goals

- This repository should not become the generic routing protocol.
- This repository should not store full Apple transcript text.
- This repository should not depend on a specific agent provider or runtime service.
